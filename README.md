# Тема 6. HTTP-запити і взаємодія з бекендом. Домашня робота

Цей проєкт є реалізацією веб-застосунку для пошуку зображень за ключовим словом з використанням публічного API [Pixabay](https://pixabay.com/api/docs/). Проєкт створено за допомогою модульного підходу для написання коду на *JavaScript* і зібрано інструментом [Vite](https://vitejs.dev/).

## Технологічний стек
- **HTML5**, **CSS3** - з використанням CSS-змінних та **Flex/Grid**;
- **JavaScript** - **ES6+** модулі, **Promises**, `async`/`await`;
- [**Vite**](https://vitejs.dev/) - збирач проєкту згідно вимог до ДЗ;
- [**Axios**](https://axios-http.com/) - для здійснення HTTP-запитів;
- [**iziToast**](https://izitoast.marcelodolza.com/) - для сповіщень та обробки помилок;
- [**SimpleLightbox**](https://simplelightbox.com/) - для перегляду зображень у модальному вікні-галереї;
- [CSS-Loader](https://cssloaders.github.io/) - індикатор завантаження... просто гарне дизайнерське рішення для інтерфейсу 😎.

## Опис роботи
Додаток дозволяє користувачеві ввести ключове слово у форму пошуку та отримати галерею з **20** зображень *(за замовчуванням)*.
- Під час очікування відповіді від сервера відображається стилізований індикатор завантаження.
- Якщо за запитом нічого не знайдено, користувач отримує відповідне попередження через бібліотеку `iziToast`.
- Кожен запит за замовчуванням отримує лише першу невелику кількість зображень, а кожен новий повністю очищає попередню галерею.
- При кліку на мініатюру відкривається повнорозмірне зображення з інформацією про нього завдяки `SimpleLightbox`.

## Авторські доповнення та покращення *(Custom Enhancements)*
Під час розробки було вирішено не обмежуватися трохи сухими вимогами технічного завдання і додати низку покращень для створення трошки більш якісного продукту:

1. **Дизайн та Стилізація:**
   - Впроваджено систему CSS-змінних (`--main-color`, `--second-color`, `--third-color`) для підтримки цілісної кольорової теми згідно з попередніми уроками по CSS.
   - Замість звичайного тексту для блоку інформації (Likes, Views, Comments, Downloads) інтегровано **SVG-іконки**, які я знайшов і додав.
   - Додано плавні hover-ефекти з використанням `box-shadow` та `transform: scale()`, що робить галерею більш "живою".
   - Використано сучасну властивість `aspect-ratio: 3 / 2` для зображень, що гарантує ідеально рівну сітку карток незалежно від їх оригінальних пропорцій (замість жорсткого задання висоти).
   - Візуально допрацьовано оверлей модального вікна `SimpleLightbox` (додано `backdrop-filter: blur(2px)` та змінено кольори іконок навігації на білий для кращої контрастності).

2. **Статистика пошуку:**
   - Реалізовано додатковий функціонал виводу статистики пошуку (`showTotalAmount`). Праворуч від форми з'являється інформація про кількість завантажених карток та загальну кількість знайдених результатів (`totalHits`, `total`).

3. **Форматування чисел:**
   - Для кращого читання чисел (кількості лайків або завантажень) написано окремий метод `digitSeparatorFix()`, який використовує своєрідний лайфгак із швейцарської локалі, щоб розбивати тисячі апострофами (наприклад, 1 234 567 → 1’234’567).

4. **Вирішення багів сторонніх бібліотек:**
   - Значення для ініціалізації `SimpleLightbox` адаптовано до сучасного стандарту ES-модулів. Обійдено один незвичний баг, при якому для методу `.refresh()` у `simple-lightbox.esm.js` присутня помилка `Class constructor cannot be invoked without 'new'` і він не оновлював галерею. Це вирішено шляхом використання методу `.destroy()` та повторної ініціалізації нового екземпляра класу під час нових запитів.
